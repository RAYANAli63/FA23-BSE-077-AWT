📚 Library Management System - Complete MVC Application

Full-stack Laravel + Bootstrap implementation with REST API design

🚀 Quick Setup Instructions

Copy and paste these commands in your terminal:

# Create project
composer create-project laravel/laravel library-management-system
cd library-management-system

# Install Bootstrap
npm install bootstrap @popperjs/core

# Create models and controllers
php artisan make:model Book -m
php artisan make:model Member -m
php artisan make:controller BookController --resource
php artisan make:controller MemberController --resource
📌 Note: Don't forget to configure your .env file with database credentials before running migrations!

🎮 Controllers

📄 app/Http/Controllers/BookController.php
<?php namespace App\Http\Controllers; use App\Models\Book; use Illuminate\Http\Request; class BookController extends Controller { /** * Scaffolding note: Resource controllers automatically generate * all RESTful methods, reducing development time by eliminating * repetitive CRUD boilerplate code. */ public function index() { $books = Book::all(); return view('books.index', compact('books')); } public function create() { return view('books.create'); } public function store(Request $request) { $request->validate([ 'title' => 'required|string|max:255', 'author' => 'required|string|max:255', 'isbn' => 'required|string|unique:books', 'published_year' => 'required|integer|min:1800|max:' . date('Y'), ]); Book::create($request->all()); return redirect()->route('books.index')->with('success', 'Book created successfully.'); } public function show(Book $book) { return view('books.show', compact('book')); } public function edit(Book $book) { return view('books.edit', compact('book')); } public function update(Request $request, Book $book) { $request->validate([ 'title' => 'required|string|max:255', 'author' => 'required|string|max:255', 'isbn' => 'required|string|unique:books,isbn,' . $book->id, 'published_year' => 'required|integer|min:1800|max:' . date('Y'), ]); $book->update($request->all()); return redirect()->route('books.index')->with('success', 'Book updated successfully.'); } public function destroy(Book $book) { $book->delete(); return redirect()->route('books.index')->with('success', 'Book deleted successfully.'); } }
📄 app/Http/Controllers/MemberController.php
<?php namespace App\Http\Controllers; use App\Models\Member; use Illuminate\Http\Request; class MemberController extends Controller { public function index() { $members = Member::all(); return view('members.index', compact('members')); } public function create() { return view('members.create'); } public function store(Request $request) { $request->validate([ 'name' => 'required|string|max:255', 'email' => 'required|email|unique:members', 'joined_date' => 'required|date', ]); Member::create($request->all()); return redirect()->route('members.index')->with('success', 'Member created successfully.'); } public function show(Member $member) { return view('members.show', compact('member')); } public function edit(Member $member) { return view('members.edit', compact('member')); } public function update(Request $request, Member $member) { $request->validate([ 'name' => 'required|string|max:255', 'email' => 'required|email|unique:members,email,' . $member->id, 'joined_date' => 'required|date', ]); $member->update($request->all()); return redirect()->route('members.index')->with('success', 'Member updated successfully.'); } public function destroy(Member $member) { $member->delete(); return redirect()->route('members.index')->with('success', 'Member deleted successfully.'); } }

📋 API Documentation (REST Architecture)

📄 public/api-documentation.md
# Library Management System - REST API Documentation ## REST Architecture Explanation ### Statelessness Our application follows REST statelessness principle. Each HTTP request from the client contains all necessary information for the server to process it. The server does not store any client context between requests. Session data is not used for API endpoints; authentication (if implemented) would use tokens sent with each request. ### Idempotence - **GET** requests are idempotent: multiple identical requests return the same result without changing server state. - **PUT** requests are idempotent: updating the same resource multiple times with the same data results in the same final state. - **DELETE** requests are idempotent: after the first DELETE, subsequent DELETE requests return 404 but the resource state remains deleted. - **POST** requests are NOT idempotent: each request creates a new resource with a new ID. ## API Endpoints | URI | HTTP Method | Description | Idempotent | |-----|------------|-------------|------------| | `/books` | GET | Retrieve all books | Yes | | `/books` | POST | Create a new book | No | | `/books/{id}` | GET | Retrieve a specific book | Yes | | `/books/{id}` | PUT | Update a specific book | Yes | | `/books/{id}` | DELETE | Delete a specific book | Yes | | `/members` | GET | Retrieve all members | Yes | | `/members` | POST | Create a new member | No | | `/members/{id}` | GET | Retrieve a specific member | Yes | | `/members/{id}` | PUT | Update a specific member | Yes | | `/members/{id}` | DELETE | Delete a specific member | Yes | | `/members/{id}/borrowed-books` | GET | Get books borrowed by a member | Yes | ## URI Design Principles - All URIs use plural nouns: `/books`, `/members` - No verbs in URIs (actions are represented by HTTP methods) - Hierarchical structure shown in `/members/{id}/borrowed-books` - Resources are identified by unique IDs

🎯 Final Setup Commands

# Run these commands after copying all files:

# 1. Install PHP dependencies
composer install

# 2. Copy environment file
cp .env.example .env

# 3. Generate application key
php artisan key:generate

# 4. Configure database in .env file
# DB_DATABASE=library_db
# DB_USERNAME=root
# DB_PASSWORD=

# 5. Run migrations
php artisan migrate

# 6. Install NPM dependencies
npm install

# 7. Compile assets
npm run build

# 8. Start the server
php artisan serve

# 9. Open browser and visit: http://localhost:8000
